# WhatToEat - 每日食谱推荐应用

## 1. 项目概述

**项目名称：** WhatToEat

**项目类型：** Windows 桌面应用程序

**核心功能：** 随机推荐一道菜，通过用户的"喜欢/不喜欢"反馈逐步学习并优化推荐结果，支持云端数据同步。

**目标用户：** 个人用户，帮助解决"今天吃什么"的决策困难。

---

## 2. 用户故事

### 2.1 核心用户故事

| 编号 | 作为... | 我想要... | 以便... | 优先级 |
|------|---------|-----------|---------|--------|
| US-01 | 个人用户 | 点击"推荐"按钮获得一道随机菜品 | 快速决定今天吃什么 | P0 |
| US-02 | 个人用户 | 对推荐结果点击"喜欢"或"不喜欢" | 让应用学习我的口味偏好 | P0 |
| US-03 | 个人用户 | 查看历史推荐记录 | 回顾我之前吃过什么 | P1 |
| US-04 | 个人用户 | 在不同 Windows 设备上访问我的偏好数据 | 跨设备同步使用 | P1 |

### 2.2 功能性需求

**FR-01：随机推荐**
- 系统从菜品数据库中随机选择一道菜进行推荐
- 避免短时间内重复推荐同一道菜

**FR-02：用户反馈**
- 用户可以对推荐的菜品选择"喜欢"或"不喜欢"
- 反馈数据将用于学习用户偏好

**FR-03：偏好学习**
- 系统根据用户的正负反馈记录分析口味偏好
- 偏好维度可包括：菜系、口味（辣/甜/咸等）、食材类型

**FR-04：云端同步**
- 用户数据（偏好设置、历史记录）同步至云端
- 跨设备访问时保留学习到的偏好

---

## 3. 非功能性需求

### 3.1 性能
- 推荐响应时间 < 1秒
- 应用启动时间 < 3秒

### 3.2 可用性
- 界面简洁，操作直观
- 单次推荐流程不超过 2 次点击

### 3.3 数据安全
- 用户数据在传输过程中加密
- 云端数据存储需考虑隐私保护

---

## 4. 技术方案

### 4.1 技术栈选型

| 层级 | 技术选择 | 理由 |
|------|----------|------|
| **桌面框架** | Tauri (Rust + Web) | 原生性能、轻量级打包、Windows 支持良好，比 Electron 更节省资源 |
| **前端** | React + TypeScript | 生态成熟，开发效率高，类型安全 |
| **样式** | Tailwind CSS | 快速构建现代化 UI |
| **后端/云服务** | Firebase | 免费额度充足，实时数据库 + Auth 一体化，适合个人项目 |
| **数据库** | Firebase Firestore | NoSQL 文档数据库，适合存储用户偏好和历史记录 |
| **认证** | Firebase Authentication | 支持邮箱/Google 登录，开箱即用 |

### 4.3 数据模型

**用户文档 (users/{userId})**
```
{
  id: string
  email: string
  createdAt: timestamp
  preferences: {
    likedCuisines: string[]      // 喜欢的菜系
    dislikedCuisines: string[]   // 不喜欢的菜系
    likedIngredients: string[]   // 喜欢的食材
    dislikedIngredients: string[] // 不喜欢的食材
  }
}
```

**历史记录 (users/{userId}/history/{recordId})**
```
{
  id: string
  dishId: string
  dishName: string
  feedback: "like" | "dislike"
  recommendedAt: timestamp
  feedbackAt: timestamp
}
```

**菜品数据 (dishes/{dishId})** - 内置基础数据
```
{
  id: string
  name: string
  cuisine: string          // 菜系：川菜、粤菜、日料、西餐等
  tags: string[]           // 标签：辣、甜、清淡、肉食、素食等
  ingredients: string[]    // 主要食材
  imageUrl?: string
}
```

### 4.4 推荐算法思路

1. **基础随机选择**：从菜品库随机选择一道未在近 N 次推荐中出现过的菜
2. **偏好加权**：根据用户历史反馈，提升/降低特定菜系的权重
3. **探索与利用平衡**：90% 按偏好推荐，10% 探索新菜品避免信息茧房

---

## 5. 菜谱数据

初期将内置一份包含 50-100 道菜的基础菜品数据集，涵盖：
- 中餐（川、粤、湘、鲁、苏、浙、闽、徽）
- 日料
- 西餐（意式、法式、美式）
- 快餐/简餐

每道菜包含：名称、菜系、口味标签、主要食材。
